# 🛒 SunCart - Seasonal B2C eCommerce Platform

SunCart is a modern, high-performance, and fully responsive seasonal eCommerce web application designed specifically for showcasing and selling summer collections. Built with user experience and speed in mind, it features secure authentication, interactive sliders, and clean product discovery workflows.

🚀 **Live Link:** [sun-cart-hdlk.vercel.app](https://sun-cart-hdlk.vercel.app)

---

## ✨ Core Features

* 🏔️ **Dynamic Hero Slider:** Immersive visual presentation featuring interactive summer sale banners and a dedicated "Hot Deals 🔥" promotional slider.
* 🔒 **Robust Authentication:** Secure login and registration powered by Better Auth, providing smooth integration for both standard Email/Password pairs and social Google Login providers.
* 🛡️ **Protected Routes:** Context-aware security ensuring product detail deep-dives and user account profiles remain strictly restricted to authenticated individuals.
* 🔍 **Product Discovery:** Seamless product discovery layout presenting ratings, responsive pricing badges, and item categorization filters (e.g., Summer Outfits, Skincare).
* 👤 **User Profile Management:** A dedicated "My Profile" space enabling logged-in users to update personal data, account names, and profile avatars instantly.
* 📱 **Fluid Responsive Layout:** Tailored and pixel-perfect design structure completely optimized across mobile, tablet, and high-resolution desktop screen boundaries.

## ⚙️ Local Setup & Installation

Follow these quick commands to install and spin up SunCart natively within your developer workspace:

### 1. Clone the Repository

```bash
git clone [https://github.com/shakibul-islam-Dev/SunCart.git](https://github.com/shakibul-islam-Dev/SunCart.git)
cd SunCart

```

### 2. Install Project Packages

```bash
npm install

```

### 3. Setup Local Environment Variables

Produce a `.env.local` context within your root structure and fulfill your active cloud access credentials:

```env
MONGODB_URI=your_mongodb_connection_string
BETTER_AUTH_SECRET=your_better_auth_secret_key
NEXT_PUBLIC_APP_URL=http://localhost:3000

```

### 4. Execute Development Instance

```bash
npm run dev

```

Point your standard web browser engine tracking location to [http://localhost:3000](http://localhost:3000).
